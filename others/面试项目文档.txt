==================================================
项目文档：空间数据安全追踪与管理系统
==================================================

1. 项目概述 (Project Overview)
--------------------------------------------------
- 项目名称：空间数据安全追踪与管理系统
- 项目总结：一个集成了矢量/栅格空间数据管理、数字水印嵌入与提取、以及多级用户权限与审批流的 Web 系统，旨在保障空间数据的安全分发与版权追踪。

2. 技术栈 (Technology Stack)
--------------------------------------------------
- 后端：Python, Flask, Flask-RESTful, SQLAlchemy, Flask-JWT-Extended
- 前端：Vue.js, Vite, Axios
- 数据库：
    * PostgreSQL (配合 PostGIS 扩展): 存储核心的矢量和栅格空间数据。
    * MySQL: 存储用户信息、权限、审批记录等业务数据。
- 核心库：GeoPandas, Rasterio, Pillow, OpenCV, PyQRCode

3. 系统核心功能与架构
--------------------------------------------------
3.1. 模块化架构：
    - 前端 (Frontend): 负责数据可视化、用户交互、审批流程展示。
    - 后端 (Backend): 提供 RESTful API 接口，处理业务逻辑、算法调用和数据库交互。
    - 数据库 (Database): 采用多数据库绑定策略，分离存储空间数据和业务数据。

3.2. 核心功能详解：
    - 统一数据视图：通过一个接口 (/api/data_viewing)，根据 dataType 参数动态查询并返回矢量或栅格数据列表，简化前端调用。
    - *** 面试亮点 *** 数字水印技术：
        * 矢量数据水印：讲解如何通过微调坐标点的方式，将二维码（二值化矩阵）信息隐写到 Shapefile 文件中，同时保证视觉上的不可察觉性。
        * 栅格数据水印：讲解如何使用 LSB (Least Significant Bit) 算法，将水印信息嵌入到图像的最低有效位，实现高容量的隐写。
        * 统一提取接口：设计一个能根据数据类型自动调用相应提取算法的接口。
    - 多级审批流：实现一个两级审批系统（管理员1初审，管理员2复审），用于控制核心数据的分发。
    - 基于角色的访问控制 (RBAC): 定义管理员和普通员工角色，通过 JWT (JSON Web Token) 实现用户认证和接口权限控制。

4. 技术挑战与解决方案
--------------------------------------------------
*** 面试亮点 ***
- 挑战1：异构数据库管理
    * 问题：如何在同一个 Flask 应用中同时操作 PostgreSQL 和 MySQL？
    * 解决方案：利用 SQLAlchemy 的 `SQLALCHEMY_BINDS` 配置和模型中的 `__bind_key__` 属性，将不同的数据模型绑定到不同的数据库引擎，实现对多数据库的无缝操作。

- 挑战2：复杂算法的实现与调试
    * 问题：矢量水印算法涉及复杂的几何操作和精度控制，如何确保其可逆性和鲁棒性？
    * 解决方案：深入研究并实现了坐标微扰算法，通过编写单元测试 (unittest) 和计算 NC (归一化相关系数) 值来科学地验证算法的有效性和可逆性。

- 挑战3：全栈调试效率
    * 问题：前端数据显示异常，如何快速定位是前端逻辑、API接口还是后端数据库的问题？
    * 解决方案：采用分层调试策略。首先使用浏览器开发者工具的“网络”面板检查 API 请求和响应是否正确；然后在后端 API 接口中添加详细的日志 (print 或 logging)，追踪代码执行路径和变量状态，最终定位问题根源。
